# v1.13 / v2.1 Main Entrypoint And Workflow Cleanup Status

Status: implemented in public workflow package.

Summary:

- Added `main_pyczsh.py` as the recommended root-level executable.
- Added `pyCSH_Zn/workflow.py` to run generation, validation, optional LAMMPS input construction, optional static relaxation, and optional quasi-static diagnostics in one Python process.
- Changed recommended default output from `pyCSH_Zn/output_Y/` to `output_pyczsh/` relative to the current working directory.
- Added unified CLI parameters for target composition, Zn count, site mode, Q1/Q2b target ratio, seeds, batch size, output path, optional LAMMPS inputs, optional static relaxation, optional quasi-static diagnostics, clean data export, and workers.
- Retained legacy scripts under `pyCSH_Zn/examples/` for compatibility.

Scientific boundaries:

- CementFF4-Zn force-field parameters were not modified.
- Validation semantics were not changed.
- The 2.5 A Zn-O coordination threshold was not relaxed.
- The old `mixed_Q1_Q2b_Zn` site type was not enabled.
- Finite-temperature MD was not introduced.
- No `md_ready_candidate` label is introduced.
- No final elastic constants or production mechanical properties are claimed.
- Q1 mechanics and multi-Zn mechanics remain opt-in.

Known limitations:

- Target Ca/Si and Zn/Si remain target-window values, not exact guaranteed compositions.
- Actual Ca/Si and Zn/Si must be read from summary outputs.
- `q1_q2b_ratio` is a target allocation ratio; actual ratio may differ because of integer Zn counts, site availability, and validation.
- LAMMPS relaxation and quasi-static diagnostics are opt-in and may dominate runtime.
