# v2.1.2 periodic framework recenter status

Version: `v2.1.2-periodic-framework-recenter`

This update adds default-enabled largest-gap-to-boundary periodic framework
recentering before final internal `.data` export and validation.

The recentering step:

- uses framework atoms as the reference for finding the largest periodic gap;
- applies one equivalent fractional shift to all atoms;
- wraps all atoms back into the same periodic cell;
- preserves cell parameters, atom IDs, atom types, charges, bonds, angles, and
  `CS-Info`;
- writes `internal/periodic_recenter_summary.json` for each model.

This is a periodic representation and visualization convenience fix. It does
not modify CementFF4-Zn force-field parameters, Zn motif definitions, the 2.5 A
Zn-O coordination threshold, validation semantics, or the opt-in LAMMPS
relaxation and small-strain diagnostic workflow.

Use `--no-recenter` to skip the step for debugging or comparison with the
original coordinate representation.
