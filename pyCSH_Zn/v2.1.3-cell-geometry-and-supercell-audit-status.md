# v2.1.3 cell geometry and supercell audit status

Version: `v2.1.3-cell-geometry-and-supercell-audit`

The v2.1.2 periodic recentering step is retained, but the newest OVITO images
show that the visible empty-box problem is no longer safely explained as only an
atom-wise wrapping or periodic cut-location issue. If the framework still
appears confined to a small corner after recentering, the likely root-cause
classes are:

- oversized or misinterpreted cell geometry;
- incomplete supercell or brick translation population;
- triclinic export or visualization interpretation mismatch;
- erroneous atom dropping or deduplication.

This update adds per-model audit files:

- `framework_occupancy_summary.json`
- `cell_geometry_summary.json`
- `brick_placement_summary.json`
- `dedup_audit_summary.json`

The audits are written before validation results are summarized and do not
modify CementFF4-Zn force-field parameters, Zn motif definitions, the 2.5 A
Zn-O coordination threshold, validation semantics, atom IDs, atom types,
charges, bonds, angles, or `CS-Info`.

When `--export-clean-data` is used, an additional
`model_XXXX.visual_orthogonal.clean.data` file may be written for OVITO-only
diagnosis. That file is not internal validation data and does not preserve the
original triclinic periodic cell.
